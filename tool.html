AlteR

An iOS modelling editor in 3D.

AlteR. (domAin modeLling augmenTEd Reality)

AlteR is a native iOS app which is using the technology ARKit for creating and manipulating the models using AR. The DSL developper will define an abstract syntax, an AR concrete syntax and 2D or 3D visualization objects, on a desktop computer. The syntaxes have to be serialized in JSON files and the visualization objects have to use the SCN format which is the Apple's SceneKit Scene format. These files are then uploaded on a remote server.

The DSL users will access the list of meta-models from the server via their iPhones and iPads using our mobile modelling environment app and they will select the meta-model of their choice, retrieve it along with the associated AR syntax and visualization objects, and they will be able to start modelling and placing element in the real world.

During the modelling session, the DSL users will be able to save their model either locally or remotely. ARKit offers a functionality for persisting the AR world map and recover the entire session later on. Every time a AR session is started, ARKit sets the world origin at the position of the camera of the device, all the placed anchors of the session get their position vector relative to this world origin. When the session is saved, a snapshot is taken, and the origin along with its anchors are saved. Thus, the DSL users will be able to load pre-existing models from the server or their local storage instead of creating a new one, and the saved anchors will be placed at their saved position as soon as the user will point the camera at the same location of the snapshot.

Modelling. An AR Editor

The screenshot on the right is the AlteR editor canvas. It shows the main screen of the mobile app. Label 1 corresponds to the list of items that can be placed in the real world and are named after the classes of the meta-model.

Label 2 points at a 3D node and depicts the list of interaction available:

  • Tap is used to place an item at the tapped location after selecting one in the list of Label 1,

  • Swipe allows to change the visual representation of a node if there are different one for the same ARNode,

  • Pinch permits to resize an item,

  • Long press an item will display its attributes and allows to edit them or to delete the item,

  • Pan moves the node in the world and change its position.

Note that the ARNode is associated with an ARAnchor and this is that later one that is serialized along with all the data about the ARNode. Label 3 shows the displayed attributes of a node as a key-value list. Finally, there are two buttons at the top to either save the model (Label 4) in order to work on it again later or to quit de modelling session (label 5).
